mysql过滤查询（以字母开头，以数字开头，非数字开头，非字母开头）

– 不是以数字开头
select * from mot_terms where name not REGEXP '^[0-9]'
– 不是以字母开头
select * from mot_terms where name not REGEXP '^[a-zA-Z]'

– 已数字和特殊字符开头
select * from mot_terms where name REGEXP '^[@#$%&0-9]'

utf8？utf8mb4!

MySQL的utf8编码只支持3字节的数据，而emoji表情数据是4个字节的字符，导致内容无法保存或乱码
可以对4字节的字符进行编码存储，然后取出来的时候，再进行解码。但是这样做会使得任何使用该字符的地方都要进行编码与解码。
utf8mb4编码是utf8编码的超集，兼容utf8，并且能存储4字节的表情字符。
采用utf8mb4编码的好处是：存储与获取数据的时候，不用再考虑表情字符的编码与解码问题。

so 更改数据库的编码为utf8mb4
修改配置文件my.cnf

[mysqld]
character-set-server=utf8mb4
 
[mysql]
default-character-set=utf8mb4
 
[client]
default-character-set=utf8mb4

重启mysql服务

tips：utf8mb4的最低mysql版本支持版本为5.5.3+

检查

SHOW VARIABLES WHERE Variable_name LIKE 'character_set_%' OR Variable_name LIKE 'collation%';

例如

Variable_name | Value
character_set_client | utf8mb4
character_set_connection | utf8mb4
character_set_database | utf8mb4
character_set_filesystem | binary
character_set_results | utf8mb4
character_set_server | utf8mb4
character_set_system | utf8
collation_connection | utf8mb4_unicode_ci
collation_database | utf8mb4_unicode_ci
collation_server | utf8mb4_unicode_ci

重要

系统变量 | 描述
character_set_client ｜ utf8mb4
character_set_connection ｜ utf8mb4
character_set_database ｜ utf8mb4
character_set_results ｜ utf8mb4

SQL

show create table <表名>;

更改数据库编码：ALTER DATABASE <数据库名> CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

更改表编码：ALTER TABLE <表名> CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
